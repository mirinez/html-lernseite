1) Überschriften & Absätze:

Überschriften strukturieren eine Seite: <h1> bis <h6>.
Absätze schreibt man mit <p>.

Beispiel

Unterpunkt (h4)

Das ist ein Absatz (p). Hier kann normaler Text stehen.

Tipp: Normalerweise benutzt man nur ein <h1> pro Seite (Haupttitel).

2) Textformatierung:

Diese Tags ändern die Darstellung von Text:

<b> = fett (nur optisch)
<i> = kursiv (nur optisch)
<u> = unterstrichen
<mark> = markiert
<small> = kleiner Text
<sup> = hoch: 102
<sub> = tief: H2O

Semantik (besser für Bedeutung):
<strong> = wichtig (Bedeutung), <em> = betont (Bedeutung).

3) Bild:

Bilder bindet man mit <img> ein.
src = Pfad/URL zur Datei,
alt = Beschreibung (wichtig für Barrierefreiheit),
loading="lazy" = Bild lädt erst später (wenn es sichtbar wird).

Wenn das Bild nicht erscheint: Name/Ordner prüfen (Groß-/Kleinschreibung!).
Tipp: Du kannst die Bildgröße über CSS (.img-small) ändern.

4) Listen:

Es gibt drei wichtige Elemente:
<ul> = ungeordnete Liste (Punkte)
<ol> = geordnete Liste (Nummern)
<li> = ein Listeneintrag

ul

  • HTML
  • CSS
  • JavaScript

ol

  1. Planen
  2. Schreiben
  3. Testen

5) Tabelle:

Tabellen sind für Daten in Zeilen/Spalten.
<tr> = Tabellenzeile,
<th> = Kopfzelle (Überschrift),
<td> = Datenzelle.

Name Nachname
Míriam Domínguez Martínez
Sayara Domínguez Martínez

Hinweis: Tabellen nutzt man für Daten — nicht für Seitenlayout.

6) Video:

<video> spielt Videos ab.
controls = Steuerelemente anzeigen (Play/Pause),
muted = stumm starten,
width = Breite des Players.

Tipp: autoplay wird oft blockiert, wenn muted nicht gesetzt ist.

7) Audio:

<audio> spielt Musik/Sound ab.
controls = Steuerelemente anzeigen,
source = Quelle + Format.

Tipp: Du kannst mehrere <source> angeben (z. B. mp3 und ogg).

8) Formular (Form) + Label:

Formulare sammeln Daten.
<form> = Formular-Container,
<label> = Beschriftung,
<input> / <select> / <textarea> = Eingabefelder.

required = Feld muss ausgefüllt werden.

type="email" prüft grob das E-Mail-Format.

method="get" = Daten in der URL (Tests).
method="post" = Daten im Request-Body (typisch bei echten Formularen).

9) Details/summary (aufklappbar):

<details> ist ein aufklappbarer Bereich, ohne JavaScript.
<summary> ist der klickbare Titel.

Klick mich (summary)

Jetzt siehst du den Inhalt in details.

10) Progress (Fortschritt):

<progress> zeigt einen Fortschrittsbalken.
value = aktueller Wert, max = Maximum.

70%

Beispiel: 70 von 100

11) Favicon (Browser-Icon):

Das Favicon ist das kleine Symbol, das im Browser-Tab angezeigt wird. Es wird im <head>-Bereich mit dem Element <link> eingebunden.

So wurde es in dieser Seite eingebunden:

Im <head>-Bereich: 

        <link rel="icon" type="favicon" href="favicon.jpg">

⚠ Hinweis: type="favicon" ist kein offizieller Standard. Viele Browser akzeptieren es trotzdem, korrekt wäre jedoch: 

        <link rel="icon" type="image/jpeg" href="favicon.jpg">

Wenn die Datei ein PNG ist: 

        <link rel="icon" type="image/png" href="favicon.png">

Am professionellsten ist meistens das Format .ico: 

        <link rel="icon" href="favicon.ico">

Die Datei sollte sich im selben Ordner wie die HTML-Datei befinden oder der korrekte Pfad muss angegeben werden.

12) Das "ingframe" Element - Externe Inhalte nahtlos einbetten:

Ein "iframe" ermöglicht es, eine andere Webseite oder ein Dokument innerhalb der aktuellen Seite anzuzeigen. Es ist wie ein Fenster zu einer anderen Seite, das nahtlos in dein Layout integriert werden kann. Mit dem "iframe"-Element kannst du beispielsweise eine Karte, ein Video oder sogar eine andere Webseite direkt auf deiner Seite einbetten, ohne dass der Benutzer die Seite verlassen muss.

Beispiel: Einbetten einer Google Map

In diesem Beispiel wird eine Google Map von Berlin eingebettet. Du kannst die "src"-URL anpassen, um andere Inhalte einzubetten, z.B. YouTube-Videos oder andere Webseiten.

Wichtige Attribute:

  • src: Die URL der Seite oder des Dokuments, das eingebettet werden soll.
  • width und height: Bestimmen die Größe des iframes.
  • style="border:0;": Entfernt den Rahmen um das iframe (optional).
  • allowfullscreen: Erlaubt das Vollbildmodus (z.B. für Videos).
  • loading="lazy": Verzögert das Laden des iframes, bis es sichtbar ist (gut für Performance).
  • referrerpolicy: Steuert, welche Referrer-Informationen gesendet werden (Sicherheit).

Tipp: Achte darauf, dass die eingebettete Seite das Einbetten erlaubt (manche Seiten blockieren dies aus Sicherheitsgründen).
