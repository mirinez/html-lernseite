1) Grundstruktur eines HTML-Dokuments

HTML ist eine Markup-Sprache. Sie beschreibt die Struktur von Inhalten – nicht das Design.

Ein HTML-Dokument besteht aus: html → Wurzelelement head → Metadaten body → sichtbarer Inhalt

Das Attribut lang="de" hilft Screenreadern und Suchmaschinen bei der Sprachzuordnung.

Diese gesamte Seite ist ein Beispiel für diese Struktur.

2) Semantik: Bedeutung statt Aussehen

Semantik bedeutet, dass HTML-Elemente eine inhaltliche Bedeutung tragen.

Beispiel: <strong> signalisiert Wichtigkeit. <em> signalisiert Betonung.

Das unterscheidet sich von <b> oder <i>, die nur visuelle Effekte sind.

3) Bilder & Performance

Das <img>-Element ist ein leeres Element. Es benötigt kein schließendes Tag.

Wichtige Attribute: src → Bildquelle alt → Alternativtext (Barrierefreiheit!) loading="lazy" → Performance-Optimierung

4) Tabellen – Datenstruktur

Tabellen strukturieren Daten in Zeilen und Spalten.

tr → Tabellenzeile th → Kopfzelle td → Datenzelle

Für komplexe Tabellen existieren: thead, tbody, caption.

5) Formulare – Datenübertragung

Formulare senden Daten an einen Server.

action → Zieladresse method="get" → Daten in URL method="post" → Daten im Request-Body

Nur Felder mit einem name-Attribut werden übertragen.

6) Multimedia in HTML5

Seit HTML5 sind Video und Audio nativ integriert. Früher benötigte man Plugins wie Flash (heute obsolet).

7) Interaktive Elemente ohne JavaScript

Das Element <details> ermöglicht aufklappbare Inhalte. Ideal für FAQs.
